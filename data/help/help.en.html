FL Lingo Help

FL Lingo is a relocalization tool for Freelancer mods. It compares your current game install with a reference install, restores known vanilla text automatically, exports unknown mod text for external translation, imports the translated result, and writes the selected changes back into the game DLLs.

Main use case: your game with an English mod on the left, and a clean localized Freelancer install on the right.

Main workflow

  1. Select Game with mod / current game.
  2. Select the Reference install / comparison game.
  3. Click Compare with reference.
  4. Export mod-only entries if the mod contains new English text.
  5. Translate the export externally.
  6. Import the translated file back into FL Lingo.
  7. Review the preview and click Apply translations.

What the statuses mean

  • Auto transferable: FL Lingo found a matching text in the reference install and can apply it safely.
  • Already in target language: source and reference already match.
  • Manually translated: the entry contains a custom imported or edited translation.
  • Mod-only content: the mod added a new entry with no target-language match in the reference install.

Editor tab

The Editor tab is the secondary workflow. It focuses on missing translations first, so you can work through open entries quickly. Use it when you want to make manual corrections without exporting everything again.

  • The right text box is editable.
  • Save edit stores a manual translation in the current project.
  • Reset manual edit removes the manual override and falls back to the automatic/reference state.

Export and import

Export mod-only creates a JSON exchange file for entries that still need translation. This is the primary workflow for new mod content.

  • You can send that file to an external translator or translate it yourself.
  • After editing, use Import translation to load the translated texts back into FL Lingo.
  • Known terminology from the active target-language terminology file is included and used for suggestions.

DLL analysis

The DLL Analysis tab shows the safest write strategy per DLL:

  • safe to replace with target DLL: the full DLL can be copied from the reference install.
  • partially relocalizable only: only matching entries will be patched.
  • not safe to replace: FL Lingo keeps its hands off the full DLL and only applies safe entry-level changes.

Projects

Use Save project to store your current session as a .FLLingo file. This includes:

  • selected install paths
  • language pair
  • paired catalog state
  • manual translations
  • DLL analysis data

Backups and safety

FL Lingo modifies Freelancer resource DLLs. Always test against a copy of the game if you are unsure.
  • A backup is created automatically before translations are applied.
  • Use Restore backup... if you want to revert written DLLs.
  • If the required resource toolchain is missing, install it from the Settings menu first.

Terminology

FL Lingo uses one terminology file per target language. You can open it from Settings - Open terminology....

  • Files live in data/terminology.<lang>.json.
  • Known faction and role names can be translated consistently from there.
  • The active terminology depends on the selected target language.

Languages and themes

  • Supported UI languages: English, German, French, Spanish, Russian.
  • Supported themes: light, dark, high contrast.
  • Language and theme are stored persistently.
